# Trading Bot

Автоматизированный трейдинг-бот для линейных USDT-инструментов Bybit. Код ориентирован на бумажный режим и безопасный запуск по умолчанию.

## Быстрый старт

1. Создайте виртуальное окружение и установите зависимости:
   ```bash
   python -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```
2. Скопируйте пример `.env`:
   ```bash
   cp .env.example .env
   ```
3. Заполните `BYBIT_API_KEY` и `BYBIT_API_SECRET`. Для уведомлений укажите `TELEGRAM_TOKEN` и `TELEGRAM_CHAT_ID` (необязательно).
4. Проверьте окружение:
   ```bash
   python sanity_check.py
   ```

## Безопасный запуск

- Бумажный режим (по умолчанию безопасный):
  ```bash
  python main.py paper
  ```
- Реальный режим (требует явного подтверждения):
  ```bash
  python main.py real --yes
  ```
- Быстрый разовый прогон (одна итерация цикла):
  ```bash
  python main.py paper --once
  ```
- CI/автопроверка (форсирует SAFE_MODE и PAPER_MODE, отключает Telegram):
  ```bash
  python main.py paper --ci --once
  ```

В режиме CI логирование ускорено, SAFE_MODE и PAPER_MODE всегда включены. Флаг `--unsafe` доступен только в реальном режиме без `--ci`.

### Запуск через TradeApp

- Меню «Запуск» в TradeApp даёт быстрые команды **Старт PAPER** и **Старт REAL**. Для реального режима перед запуском открывается диалог подтверждения с переключателем SAFE_MODE.
- Кнопка с иконкой «?» вызывает справочник по управлению ботом: быстрый старт, горячие клавиши, описание кнопок и переменные окружения.
- При запуске из TradeApp автоматически выставляются переменные `PAPER_MODE`, `SAFE_MODE`, `LOG_JSONL` и UTF-8 кодировка подпроцесса.

## APK Manager и защита REAL-режима

Экран APK Manager (флаг `ENABLE_APK_MANAGER`) доступен из главного меню и верхней панели TradeApp. Все действия логируются в `logs/apk_manager.jsonl`.

### Возможности

- Загрузка APK из локального файла, произвольного URL или последнего GitHub Release.
- Проверка SHA256 скачанного APK и отображение текущего файла.
- Раздача через встроенный HTTP-сервер с одноразовыми токенами, Basic Auth (при включении) и авто-стопом.
- Генерация QR-кода для удобного скачивания с мобильного устройства.
- Список устройств ADB и установка APK через `adb install` (USB или Wi-Fi).

### Переменные окружения

```env
# APK manager
ENABLE_APK_MANAGER=1
APK_DEFAULT_PORT=8787
APK_DEFAULT_AUTOSTOP_MIN=15
APK_BIND_ALL=0
APK_ENABLE_BASIC_AUTH=0
APK_BASIC_AUTH_USER=tradeapp
APK_BASIC_AUTH_PASS=changeme
TRUSTED_IPS=127.0.0.1,192.168.1.0/24
ONE_TIME_APK_LINK=1

# Безопасность REAL-режима
ENABLE_PIN_FOR_REAL=1
PIN_HASH=<sha256(PIN)>
ENABLE_TG_2FA=0
TG_2FA_TTL=300
```

SAFE_MODE остаётся включённым по умолчанию, ссылки могут быть одноразовыми, а сервер по умолчанию слушает только выбранный интерфейс.

### Ручной чек-лист

1. Включите `ENABLE_APK_MANAGER=1`, перезапустите TradeApp и откройте вкладку «APK Manager».
2. Выберите или скачайте APK, нажмите «Проверить SHA256» и убедитесь, что хэш совпадает.
3. Запустите раздачу, сгенерируйте QR и скачайте APK с мобильного устройства. При `ONE_TIME_APK_LINK=1` ссылка должна инвалидироваться после первой загрузки.
4. Подключите устройство по USB и выполните установку через ADB — в логе появится «ADB install success».
5. Попробуйте запустить REAL без заданного PIN — операция должна быть заблокирована. После установки `PIN_HASH` (и включения `ENABLE_TG_2FA`, если нужно) ввод PIN + код из Telegram разрешит запуск.

## Управление во время работы

- `Space` в UI (`trade_app.py`) отправляет `{"toggle_pause": true}` в `control.json` — bot переключает паузу входов и сохраняет состояние в `control_state.json`.
- Кнопки «Пауза» и «Возобновить» отправляют `{"pause_entries": true|false}`.
- Кнопка «Паника» посылает `{"panic_close": true}` и инициирует закрытие позиций.
- Меню «Запуск» доступно из любой вкладки, а кнопка «?» открывает подсказки по интерфейсу.

## Проверки и CI

- `python sanity_check.py` — статическая проверка окружения, моделей и директорий.
- GitHub Actions workflow `.github/workflows/bot-ci.yml` запускает sanity check и бумажный smoke-тест (`python main.py paper --ci --once`).

## Журналы и данные

Логи и данные сохраняются в директории, заданной переменной `DATA_ROOT` (по умолчанию `./data`). JSONL-лог цикла определяется переменной `LOG_JSONL`. Скрипт `sanity_check.py` создаёт недостающие директории автоматически.

### Сессии запуска

Каждый старт бота попадает в каталог `data/sessions/<YYYYMMDD_HHMMSS>/` и содержит:

- `meta.json` — исходные параметры запуска (режим, пары, git SHA, окружение, лимит свечей).
- `log.txt` — полный stdout/stderr, зеркалированный из консоли.
- `log.jsonl` — симлинк или копия основного цикла (`LOG_JSONL`).
- `trades.jsonl` — сделки за сессию (фильтрованные из `log.jsonl`).
- `stats.json` — итоговая сводка (PnL, winrate, длительность, max DD, Sharpe и т.д.).
- `favorites.flag` — пустой маркер «избранной» сессии (создаётся из UI).

Индекс `data/sessions/index.json` обновляется автоматически для быстрой загрузки списков в TradeApp.

### Полезные переменные окружения

- `KLINE_HISTORY_LIMIT` — глубина истории 1m-свечей (по умолчанию 300) для расчёта EMA/ATR без повторного прогрева.
- `LOG_RU` — включает русские шаблоны логов (1 по умолчанию).
- `ROUTER_HEARTBEAT_SEC` — минимальный интервал между повторными сообщениями роутера (по умолчанию 60 секунд).
- `SAFE_MODE` / `PAPER_MODE` — управление режимами безопасности и моделирования.
